Edge GPGPU · Architecture Design

一颗可复现的
Jetson 级开放 SM

curryGPU 是一颗 clean-slate、NVIDIA-SASS 设计哲学的端侧 GPGPU：AI 张量算力 + 通用 SIMT 均衡，单 die、约 15–60W，从编译器往下设计，以编译期静态调度为能效差异点。

AI + 通用 均衡 15–60W · 单 die · LPDDR5 FPGA 验证 → 学术流片 PTX-in / 自定义 SASS-out
01

概览

positioning

对标在峰值算力与能效 TOPS/W双维上超过开源 RISC-V GPGPU Vortex 与 承影/Ventus。一句话定位：一颗可在 FPGA / 学术流片上复现的、开放/自定义的 Jetson-Thor 级 SM——NVIDIA 最新的 Jetson Thor 用的正是 Blackwell GPU，本身就是"把 Blackwell 下放到端侧"的官方样板。

明确不做数据中心专用、FPGA/学术流片不可复现的特性一律剔除：双 die NV-HBI、HBM、NVLink、机架级互连、RAS / 机密计算引擎。架构创新集中在张量数据通路、静态调度与异步供数。

04

依赖与静态调度

能效差异点

curryGPU 在能效维度超过 Vortex/Ventus 的核心赌注。GPU 把依赖跟踪交给软件——硬件不做寄存器重命名 / 记分牌 / ROB，省下的逻辑让位给执行单元与寄存器堆。每条指令的控制字段显式编码 RAW/WAR/WAW。

管线三分类与 scoreboard

管线类对象依赖处理
coupled短延迟 math / ALU / FMAlatency table + 静态填充 + reuse，通常不分配 scoreboard
redirected张量等中延迟部分 RAW 边界需 scoreboard
decoupledLSU / 异步 / barrier必须显式 scoreboard 或 depbar

控制字段三分：req_bit_set（consumer 等待）/ src_rel_sb（释放读 scoreboard，处理 WAR）/ dst_wr_sb（产生写 scoreboard，处理 RAW）。编译器流程：def-use → 虚拟 scoreboard web → 剪枝 → 物理 SB 染色 → 写 &WR/&RD/&REQ + wait token。

能效依据 · 实测控制位逻辑面积仅占寄存器堆的 0.09–0.13%，等效硬件记分牌需 5.32–7.09%，且静态调度性能更优（记分牌仅 0.97×）。源：arXiv:2503.20481 / 1804.06826。
05

张量子系统

spatial · low-precision

空间脉动阵列：延迟随 tile 增大近恒定、靠加宽 lane（而非加深流水）扩吞吐——参照 Blackwell 第五代 Tensor Core 的空间阵列行为。PE 为 DSP-free 融合点积，纯 LUT 实现便于 FPGA 复现。

张量指令族为独立资源域（类 HMMA/IMMA/QMMA/OMMA），shape 字段化、跨 group 完成，异步发射 + group scoreboard（commit_group / wait_group）收尾。

06

工具链

PTX-in / SASS-out

clean-slate 只在硬件 ISA 层，编译器层站在成熟前端上：复用 NV 前端，curryGPU 作为新后端目标（类比"为这套编译器加一代新架构"）。拿到 PTX 级 CUDA 生态兼容 + 现成静态调度生成器，把工具链风险降到最低。

开源化注记研究/复现阶段可直接参照成熟编译器；若将来要做纯开源可流片工具链，外部派生部分需 clean reimplementation——自有的指令 schema 与设计知识可复用。

对标与目标带

benchmarks
口径约定NVIDIA 标称 AI TOPS 内嵌 2:1 结构化稀疏的 2× 翻倍，对标必须折半还原稠密；NVIDIA 自身脚注确认 FP8 TFLOPs ≡ INT8 TOPs。Jetson 数为理论稠密峰值（实测约 57%），功耗为模组级 TMP（含 CPU/DLA/内存），非 GPU-only。
关键 · 数据现实Vortex 与 Ventus 的论文都未在同一配置下同时给出吞吐 + 资源 + 功耗，因此无法从公开数据算出它们的 GFLOPS/kLUT 或 TOPS/W——能效横向对比必须自行复现其 RTL、在 curryGPU 同一平台上实测。唯一有坚实面积效率代理的是 Ten-Four。

④ curryGPU 三条验收门槛

  • 面积效率（FPGA · 主验收轴）provisional：同织构 FPGA 上 超过 Ten-Four ~0.714 GFLOPS/kLUT——唯一有坚实公开基线的轴。凭 DSP-free + FP4/MX（每 LUT 有效 op 更高）目标拉开 ≥2× 余量；具体倍数待 RTL 综合证实。
  • 峰值算力 + 对 Vortex/Ventus 横向（FPGA）provisional：因二者无公开配对数据，以自行复现其 RTL、同平台实测为准；暂以文档锚点（Vortex 21.5K LUT@233MHz；Ventus 3.2 GFLOPS@100MHz/11%）为对照。
  • 能效（流片态）：无 Vortex/Ventus 硅基 TOPS/W 可比 → 目标 超过 MXDOTP 0.356 TFLOPS/W，并落在 e-GPU <28mW(16nm) 包络内；方向锚 Orin NX ~2 → Thor ~8 稠密 TOPS/W。

注：Ten-Four 的 134 GFLOPS 是理想满流水综合值（非实测 kernel），0.714 GFLOPS/kLUT 应视作理论峰值上限；跨 FPGA 织构（Intel ALM vs Xilinx 6-LUT）的 GFLOPS/kLUT 不严格 1:1。

借鉴来源

下放清单

从 Blackwell/Hopper 可下放到端侧的部分（均经多源对抗校验）。跨代依赖建模演进作为 ISA 参照：Volta（软件管理依赖系统化）→ Turing（uniform 路径）→ Ampere（异步内存）→ Hopper（异步张量 + warpgroup + fence）→ Blackwell（张量内存 + 新 MMA 通路）。

借鉴要点
低精度张量通路FP4≈2×FP8≈4×FP16，靠加宽而非加深扩吞吐
微缩放 MX / NVFP4极小硬件换 4-bit 可用精度
静态调度控制位能效最高杠杆（面积 0.09–0.13% vs 记分牌 5.32–7.09%）
空间脉动张量阵列近常数延迟、加宽扩吞吐
异步供数TMA 式 + cp.async 式，张量满吞吐必需
uniform 数据通路warp 内标量/地址计算卸出向量通路
SM 分区参照大 RF、缩 L1、几乎不做 FP64
09

ISA 与编码

指令集定稿→ isa.html

ISA 信息密度最高，已独立成文档：docs/design/isa.html。策略为借模型·定制编码——采用 NV 五层 instruction-class schema 与 21-bit 控制段语义，opcode 编号与位布局全部自定义。ISA 主体已成稿——骨架、全指令族（计算 / 访存 / 张量低精度 / 控制同步 uniform）与编码定稿，要点如下。

  • 指令字：128-bit 定长，控制段 21 + guard 4 + opcode ≈12 + 载荷 ≈91（含 descriptor/TMEM 变体）。
  • 操作数：GPR(8-bit)/URF/谓词 P·UP/SR/c[bank][off]/descriptor/TMEM-addr；6 地址空间（新增张量近存 tensor-mem）。
  • 依赖：6 物理 scoreboard，coupled/decoupled，virtual queue 资源域全定义并实现 math·AGU·SFU·MMA·TMA·XLANE·CBU·fence。
  • 分歧：独立线程调度 ITS（每 lane 独立 PC + 收敛屏障）；warp=32 锁定。
  • 张量数据流：TMA 在 global↔shared 供数；脉动阵列 PE 内累加、drain 到张量近存（见 ISA §05）。
  • 计算类：正交母机（IADD3/IMAD/LOP3/SHF/ISETP）+ modifier/LUT 合成；低精度 pack/cvt（F2FP/F2SF）+ INT8 点积 IDP4A；SFU 独立 decoupled 域（见 ISA §07）。
  • 访存类：三档搬运（同步 LD/ST · 异步 CPY · TMA bulk）+ mbarrier 跨 warp 完成；地址空间作 operand 类型（见 ISA §08）。
  • 张量 + 低精度：单 MMA 母机（.kind/.shape/idesc）+ 张量近存 LDT/STT；block-scale 闭合 MX/NVFP4 量化链（见 ISA §09）。
  • 控制 / 同步 / uniform：收敛屏障 + 屏障 / warp 协作（amax=REDUX.FMAX.ABS）+ uniform 通路；cross-lane 规范名归一（见 ISA §10）。
  • 编码定稿：91-bit 载荷单字够用（最紧 TMA 5D 余 16-bit）；relocation 保护控制段静态调度契约（见 ISA §11）。
