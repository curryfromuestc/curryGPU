01

指令字与控制段

128-bit · static control

128-bit 定长，每条指令自带 21-bit 控制段。固定骨架 37 bit（控制段 21 + guard 谓词 4 + opcode ≈12），其余 ≈91 bit 给操作数 / 立即数 / 修饰符。单发射 in-order / warp。

控制段 = 统一的软件依赖机制21-bit 控制段是一套软件可见依赖管理，覆盖固定与可变两类延迟：固定延迟（math，如 FFMA）由 stall 静态隐藏；可变延迟（内存 / SFU / 张量）由 SB 位（生产者 issue 时增、写回或读源后减）+ wait-mask 表达——同样是控制位编码，并非另一套机制。详见 §03。
02

寄存器与操作数

files · address spaces · operand types

向量与标量两套寄存器文件，6 个地址空间，operand 类型按机器语义分类——编码上是类型而非 mnemonic 后缀，验证器 / 调度器 / 反汇编器据此判断合法性与依赖。

地址空间（6 类，从近到远）

operand 类型

编码区分以下类别；descriptor 与 TMEM-addr 是张量/异步路径的关键——fragment 装不进直接寄存器列举时，用驻留在 UR 的 small-object descriptor 间接描述来源与 layout：

  • GPR / UR：普通与 uniform 寄存器（两条独立 operand path，不可混为一谈）。
  • P / UP：谓词与 uniform 谓词（guard 与指令内部谓词）。
  • SR：特殊寄存器（只读机器状态）。
  • const：c[bank][offset] 常量操作数——kernel 参数 / 常量直接入指令，省 RF 与发射。
  • descriptor：TMA descriptor（global tensor 形状/stride/tile/swizzle）、memory descriptor；驻 UR，要求 warp 内 uniform。
  • TMEM-addr：指向张量近存内部某段累加器 / fragment，不进入 GPR 分配（行 / 段 / lane 位布局见 §11）。
  • shared.remote / DSMEM：cluster 内远程 CTA 的 shared 地址 = %cluster_ctarank（远程 CTA selector）+ shared offset；LDS / STS / ATOMS / RED 与 TMA .MCAST、mbarrier .CLUSTER 据此寻址，对齐 / 越界 / 权限规则随 shared——非新增地址空间，是 shared 的 cluster 形态（见 §10）。
03

依赖与 scoreboard

software-managed dependency

依赖跟踪交给软件：硬件不做寄存器重命名 / 动态记分牌 / ROB，省下的逻辑让位给执行单元与寄存器堆。21-bit 控制段是一套统一的软件可见依赖机制，同时覆盖固定与可变两类延迟——可变延迟由控制位中的 SB 位 + wait-mask 表达，并非另一套机制。

两类延迟，同一套控制位

延迟性质 · 管线对象控制位如何处理
固定延迟 · coupledmath / ALU / FMAstall counter 静态隐藏（= 生产者延迟 − 间隔指令数）+ reuse；不分配 SB
可变延迟 · decoupled内存 / SFU / 异步 copy / TMA / 张量SB 位（issue 增 / 写回·读源后减）+ wait-mask；可用 DEPBAR.LE 显式等待

可变延迟：SB 位 + wait-mask（控制位的一部分）

编译器不知道内存 / SFU / 张量指令的执行时间，无法用 stall——这类依赖完全由控制位表达。6 个 Dependence counter（SBx，x∈[0,5]，各计到 63）：

  • dst_wr_sb（&WR，3-bit）：生产者 issue 时 +1、写回时 −1，跟踪 RAW / WAW。
  • src_rel_sb（&RD，3-bit）：读屏障——保护源操作数被本指令读完前不被后续写者覆盖（WAR）。issue 时 +1、源读取完成即 −1（生命周期是 source-read，非 writeback），比等写回早释放（内存指令可省数百 cycle）。
  • wait-mask（&REQ，6-bit）：consumer 发射前要等哪些 SB 归零，最多可同时等全部 6 个。

两种等待原语：wait-mask 等选中 SB 归零（EQ-0）；DEPBAR.LE SBx, N 对单个 SB 等到剩 ≤N 个未完成——in-flight 组的阈值等待，用于异步 copy / TMA 组（6 SBx 范围内）；MMA group scoreboard 独立于 6 SBx，由专用 MMA.WAIT / MMA.DEPBAR.LE 等待（见 §09）。SB 自增有 1 拍可见延迟——紧邻 consumer 由 stall≥2 或 yield 兜底。编译器流程：def-use → 虚拟 SB web → 剪枝 → 物理 SB 染色（6 个）→ 写 &WR/&RD + wait-mask；6 个 SB 在张量供数 + 多笔异步 copy in-flight 时提供并发度。

能效依据 · 实测控制位逻辑面积仅占寄存器堆的 0.09–0.13%，等效硬件记分牌需 5.32–7.09%，且静态调度性能更优。源：arXiv:2503.20481 / 1804.06826，适用于该两篇的测试配置。

内存一致性

弱内存序 + scoped fence：默认 .WEAK，需要时 .STRONG 按 scope 生效；scope 阶梯 CTA / cluster / GPU / SYS（cluster 随 §10 thread-block cluster 启用）。执行依赖与内存可见性分离：SB / wait 只约束指令发射 / 完成（数据就绪），不等同内存排序；跨 generic / async-proxy 的可见性由 fence（含 FENCE.PROXY.ASYNC）、sem.scope、mbarrier 的 release / acquire 决定——mbarrier complete_tx 本身不自动传递排序。

04

分歧与重聚

independent thread scheduling

warp 内分支采用独立线程调度（ITS）：每 lane 独立 PC，收敛屏障管理分歧与重聚，支持 starvation-free 前进与细粒度同步。

项取值
分歧 / 重聚收敛屏障指令（BSSY / BSYNC / BRA.DIV 式）
per-lane PC每 warp 32 个独立 PC + active mask
前进保证starvation-free，规避经典 SIMT 死锁
warp 宽度32 lane（与 NV 前端 / 张量 fragment 绑定）

warp = 32 由前端复用与张量 fragment 布局决定（shfl / vote / match、MMA fragment 的 lane 映射、bar.warp.sync 均依赖 32）。高分歧小 kernel 的 lane 利用率由 subwarp 发射在 uarch 层优化，不改 ISA warp 语义。

05

张量数据流与近存

systolic · tensor-mem · TMA

TPU 系数据流：TMA 在 global ↔ shared 间异步搬运多维 tile；脉动阵列从 shared 取输入、在 PE 内累加（output-stationary）；累加器与输出 drain 到专用张量近存，离开寄存器堆。

脉动阵列 / 张量近存TMA · uniform engine 通用存储完成同步

TMA（Tensor Memory Accelerator 式）

在 global ↔ shared 间搬运多维 tile（含 stride / swizzle / 越界策略）。走 uniform engine：发起指令占一次 issue，数据搬运与地址迭代不持续占 warp 发射槽；不经 RF staging；一条指令替代大量地址生成。完成：load 经 mbarrier complete_tx，store / reduce 经 bulk-group commit/wait + BULK-SB（见 §08）。

能力形态
tile 维度2D–5D tile（UTMALDG / UTMASTG 式）
布局选项stride + swizzle · 尾部 mask / ignore_oob · im2col
方向load / store / reduce（UTMAREDG · redOp×type×scope）
multicast.MCAST（随 cluster · §10）
完成 / 同步load→mbarrier complete_tx · store / reduce→bulk-group

张量近存（tensor-mem）

独立地址空间，LDT / STT 式专用访问（走 AGU 资源域、占物理 SBx，与 LDS/STS 平行，不新增资源域）+ TMEM-addr 操作数类。承载脉动阵列的累加器与输出 drain，让中间结果离开 RF、支持大 tile 累加；权重 / 累加初值经 shared → tensor-mem 直拷预载，不走 RF 回程。规模按阵列尺寸与 tile 在 uarch 阶段定，指令面见 §09。

计算类指令族

int · fp · cvt · sfu · pred

少而正交的工作母机（IADD3 / IMAD / LOP3 / SHF / ISETP）+ modifier / LUT 合成覆盖整数·浮点·位运算，opcode 不爆。全族归 math / coupled（latency table + stall 静态隐藏、通常不分配 scoreboard）；SFU 超越函数为 decoupled 例外（独立资源域、写 scoreboard）——同属 decoupled 的还有 cross-lane 归约（§10），不止 SFU 一处。

低精度计算路径（差异点）

  • SIMT 算术下限 F16/BF16：packed HADD2 / HFMA2 等；FP8/FP6/FP4 的乘加只走脉动阵列张量域，不在 SIMT 通路下探。
  • pack/cvt 下探到 FP4：F2FP（强制 .SATFINITE）、I2IP（INT8/INT4）、F2FP.RS（随机舍入，显式 random-bits 源）。
  • INT 点积 IDP4A / IDP2A：SIMT 通路的 INT8 4-way / INT16 2-way 点积，作小算子 fallback（不值得起脉动阵列时），与张量 MMA 互为 fallback。
  • 低精度激活：MUFU.EX2 / .TANH 的 f16x2/bf16x2 packed，张量输出就近后处理，免 f32 往返。

关键约定

  • 正交母机 + 合成：签名 / 宽度 / 方向 / 比较关系全用 modifier；IMUL = IMAD(c=RZ) 别名；不铸 *32I 独立 opcode（32-bit 立即数内嵌为 srcB 形态，每指令至多一个"大"操作数 imm32 ⊕ c[][] ⊕ URb）。
  • 进位走谓词：IADD3.X / IMAD.X / ISETP.EX 用普通谓词传 carry，不设全局 CC 寄存器。
  • 谓词面唯一：ISETP / FSETP（双谓词输出 + 布尔组合）权威定义于 PRED-IMM 族，整数 / FP 族只引用 + 各自 dtype 约束；PLOP3 为谓词 LUT。
  • SFU 对照：IEEE 级精度由编译器 MUFU 种子 + FMA 牛顿迭代 合成，不设单条 IEEE rcp/sqrt；SQRT 由 RSQ 合成。

四类可变延迟完成跟踪共享 §03 的 6 个 Dependence counter，分工固定以保证并发度：

关键约定

  • 地址空间即 operand 类型：通用 LD/ST 按地址空间选 RF / shared / global 通路，不靠 mnemonic 后缀堆叠语义。
  • cache modifier 收敛（仅普通 LD/ST）：load {.CA/.CG/.CS/.CV/.LU}、store {.WB/.CG/.CS/.WT} 与 order × scope 互斥；async bulk / atom / red / prefetch 不受此限，各自带合法的 sem.scope × cache-hint × redOp 组合。
  • atom / red 语义矩阵：op ∈ {add,min,max,inc,dec,and,or,xor,cas,exch} × type（含 int / FP / F16 / BF16）× sem.scope（CTA/cluster/GPU/SYS）；合法 / 非法组合的完整真值表于编码定稿 / 机器描述层展开——attention counter、MoE histogram、optimizer 更新所需。
  • descriptor 驻 UR：算 operand 类型，不占「大操作数」（imm32 ⊕ c[][] ⊕ URb）名额（见 §11）。
  • TMA 完成分三套：UTMALDG（global→shared）经 mbarrier complete_tx（tx-count）；UTMASTG（shared→global）与 UTMAREDG（reduce，带 redOp × type × scope）经 bulk-group commit/wait + BULK-SB 跟踪完成。bulk 支持尾部修饰 {.cp_mask, .ignore_oob, .b128} 与只等源读完的 wait.read——ragged attention / 变长 KV-cache tile 边界所需。
  • TMA 只落 shared / L1：global ↔ shared 双向；tensor-map descriptor 由 const / param 句柄入 UR；tensormap.replace 原子改写 descriptor 字段（base / dim / stride / box / swizzle），与 in-flight TMA 的 hazard 由 commit/wait 隔离——字段位布局于编码定稿层展开。
  • L2 residency 策略：createpolicy / applypriority 式 cache-hint 控制 KV-cache 驻留 vs 流式逐出（persisting / streaming / normal），挂普通 LD/ST 与 TMA 的 cache-hint 槽；窗口与策略编码于编码定稿层展开。
  • mbarrier 退出语义：ARRIVE.DROP 让线程退出 barrier 参与集（warp-specialization 部分 warp 退出），与 .ETX（expect-tx）、release / relaxed 组合。
  • DEPBAR.LE 组等待：异步 copy / TMA 组完成用它（对单 SB 等 ≤N，区别于 wait-mask 的 EQ-0）；MMA group 完成走专用 MMA.WAIT / MMA.DEPBAR.LE（group SB 独立于 6 SBx，见 §09）。
  • cluster 层：thread-block cluster 引入分布式 shared memory（DSMEM，§02）；ARRIVE.CLUSTER{.ETX,.DROP} 对远程 CTA 的 DSMEM mbarrier 执行 arrive，按 release scope 原子推进目标 phase / tx-count，消费侧 TRYWAIT 按 acquire scope 观察；TMA .MCAST multicast（UTMALDG.MCAST 多 CTA 共享 tile）、跨 CTA red 聚合、scope 阶梯 cluster（§03）一并启用。

关键约定

  • 单 MMA opcode：格式差异（F16 / FP8 / FP4 / INT8）是 .kind 而非独立 opcode；M/N/K 唯一由 idesc 给出、.shape 仅选 tile-class——对齐 §07 的 IMUL = IMAD 别名原则。
  • leader 单线程发起（SINGLETON）：MMA 为 SINGLETON 指令，由 ELECT 在 membermask 内选出唯一 leader lane（@elected guard）发描述符；验证规则 = membermask / active 内 exactly-one leader，否则 illegal。非 leader lane 在该 PC 处 guard 假、跳过，ITS 下由收敛屏障保证 warp 在 MMA 处收敛、leader 与非 leader 同步前进。累加开关 enable-acc 是 leader 的 scalar 输入谓词，不再用 per-warp UP。对齐 Blackwell tcgen05 单线程发起，与「A/B 经 shared descriptor、无 GPR fragment」自洽。
  • 累加器在张量近存内累加：A/B 经 shared descriptor 流入、无 GPR fragment；C/D 驻 TMEM-addr，经 LDT/STT 与 RF 往来。
  • 完成 = pre-fence + 专用 group scoreboard：MMA.FENCE 保证 shared 上 A/B 写在 MMA 读前可见；MMA.COMMIT 提交、MMA.WAIT N / MMA.DEPBAR.LE 等阈值。MMA group scoreboard 独立于 6 个通用 SBx，故只能由 MMA.WAIT / MMA.DEPBAR.LE 寻址——通用 DEPBAR.LE（管 6 SBx 的 copy/TMA 组）不跨等 MMA group。MMA 属 async proxy，完成可经 .MBAR 通知跨 warp 消费者。
  • TMEM 内存序独立于寄存器就绪：LDT（TMEM→GPR）的寄存器 scoreboard 不代表 TMEM 内存排序，后接 MMA / 访存前用 LDT.WAIT 显式等待（对齐 NV tcgen05.wait::ld）。
  • block-scale 闭合量化链：scale 矩阵（MX UE8M0 / NVFP4 E4M3）驻张量近存；.block 粒度（B16/B32）必须与 §10 的 amax 归约窗口（∈{16,32}）对齐，否则块内缩放与归约窗口错位。
  • 低精度权重经 CPYT 预载：shared → tensor-mem 直拷，含 CPYT.UNPACK 的 FP6（E3M2/E2M3）/ FP4 解压——NVFP4/MXFP4 权重路径所需（决策 4 低精度定位）；多 warp 共享 tile 的 multicast 随 cluster 提供（§10）。
  • MMA.SP（2:4 / 4:8 结构化稀疏）：稀疏权重 MMA（有效吞吐≈2×）；sparse metadata 选择子绑定 reuse[4] 第 4 槽（稀疏 MMA 下该槽专用、不作 reuse），脉动阵列 PE 按 metadata 跳过零元素。
  • accumulator-domain 归约：LDT.RED 对张量近存累加器直接做 row / col 段归约（attention row-max/row-sum、layernorm mean/var、per-channel amax），免 LDT→GPR→SIMT redux 往返。

关键约定

  • cross-lane 规范名唯一：VOTE / REDUX / MATCH / SHFL 定义于 warp 协作族；uniform 不铸独立 VOTEU / UREDUX，只作写 UR/UP 的 .U 变体。WARPSYNC 单一 class，mask 来源（P/UP）是操作数变体。
  • cross-lane = decoupled + 写 SB：归约延迟非定值，与 §07 的 SFU 同列 decoupled——故「SFU 是唯一 decoupled 例外」不成立。
  • amax = REDUX.FMAX.ABS：量化链第一环 owner；归约窗口 ∈{16,32}，须与 MMA .block（B16/B32）对齐。
  • 显式 membermask + active 基线：cross-lane / WARPSYNC 带 32-bit membermask 操作数声明参与 lane 子集（支持 subwarp 分组 softmax / MoE top-k）；验证基线 = 该指令 issue 时当前收敛域的 active mask（要求 membermask ⊆ active、mask 内 lane 须执行同一 collective 否则 UB），active mask 不替代显式 mask；需要时先 WARPSYNC membermask 重收敛。EXIT/KILL/RET 退出的 lane 即时从所有活跃 CBR 撤离（starvation-free 硬前置）。
  • guard[4] 只寻址 P：条件分支 guard = 3-bit P-index + 1 negate；UP-guard 由 uniform 控制 opcode（UBRA / UWARPSYNC）隐含取自 UP-file，不需第 5 bit 选 class。
  • 动态寄存器上限：SETMAXNREG.{INC,DEC} 在 warp-specialization 下让 producer warp 降寄存器、consumer warp 增寄存器以提 occupancy；SINGLETON。
  • cluster 层：thread-block cluster 引入后，scope 阶梯插入 cluster（§03）、BAR.CLUSTER 跨 CTA 同步、mbarrier .CLUSTER 远程 arrive、TMA .MCAST multicast、分布式 shared memory（DSMEM，§02）一并启用。
  • 单 FENCE opcode：ordering / 可见性由 modifier 组合（scope × proxy）；MEMBAR = FENCE 别名；scope 含 cluster（CTA / cluster / GPU / SYS，FENCE / sem.scope / atom-red / mbarrier 统一适用，覆盖 DSMEM 与 remote arrive 的 release-acquire）。FENCE.PROXY.ASYNC（TMA → shared → 脉动阵列关键路径）。
  • BRANCH_TYPE 含 BRT_BRANCHOUT：BRA.DIV / BREAK（离开收敛域）标 BRT_BRANCHOUT，区别于域内 BRT_BRANCH，供 CFG / 收敛域分析。
  • SINGLETON 机器属性：全部 CBU 同步 / 控制类 + CPY_COMMIT / UTMACMDFLUSH / MBAR.* + MMA（leader-issue，§09）显式声明 SINGLETON=true（独占发射槽）。
  • named barrier 独立：CTA 命名屏障 0–15（4-bit id 在 payload）与 6 个依赖 SB（3-bit 选择）是两套，互不挤占。
  • uniform 桥接非对称：R2UR（GPR→UR，跨 lane 归一，重）/ UR2R（UR→GPR，近免费）；ELECT（leader 选举）供 TMA 与 MMA leader-issue 发射（§05 / §09）。

编码定稿与重定位

bit budget · relocation · disasm

所有指令最宽形态单 128-bit 字编码通过：最紧形态 TMA 5D-tile 余 16 bit，不需 short-imm、不需双字。重定位与反汇编保护控制段的静态调度契约。

整字分区与大操作数口径

  • 整字分区（高位 → 低位）：控制段[21] · opcode[≈12] · guard[4] · 载荷[≈91]——编码器 / 反汇编器 / 调度器共用这一顺序（与图 1 一致）。
  • operand-form selector fsel[3]：互斥选择载荷内「大操作数」形态 imm32 ⊕ c[bank][off] ⊕ URb。
  • 「至多一个大操作数」是 operand 类别计数规则，非 bit 预算：每条指令至多一个 imm32 / c[][] / URb；GPR（8-bit）、UR 句柄（6-bit，如 descriptor）算小操作数，不占该名额。相邻 UR 对（如 DESCa/DESCb）算单个 URb 大操作数。
  • TMEM-addr 操作数：行 / 段 / lane 字段在此定位，不进入 GPR 分配域。
  • 控制段借位（sparse）：reuse[4] 第 4 槽在 MMA.SP 下重解释为 sparse-metadata 选择子（spmeta_sel），由 .SP modifier 触发、调度器不按 RF reuse 解释；属控制段语义复用，不占 91-bit 载荷。借位 / 重解释完整表于编码定稿层给出。

重定位与反汇编

  • R_CURRY_* 重定位：action-based、4-state handler，覆盖 PC-relative 分支 / 调用、global 符号（ABS）、TMA descriptor base、const-bank offset。
  • 保护控制段静态调度契约：R_CURRY_INST128 不得改写 21-bit 控制段；R_CURRY_OPFIELD 改 opcode 须满足 SAME_DOMAIN_AND_LATENCY 不变量（同资源域 + 同延迟类），否则链接期报错——否则 stall / SB 落点失效。
  • 补端侧 AI 重定位：dynamic / extern shared 窗口 base（launch 指派）、mbarrier / shared 符号、grid-constant（kernel-param：tensor base / dims / scale-factor 指针）。
  • canonical 反汇编：pseudo-op 归一（IMUL→IMAD(c=RZ) · MEMBAR→FENCE · UIMUL→UIMAD · KILL→EXIT.KILL · UDEPBAR→DEPBAR.LE）；modifier 固定打印序；round-trip 验证含控制段维度（stall / yield / SB / wait-mask / reuse）。
